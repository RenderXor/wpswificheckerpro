¬øPor que necesita permisos de ubicacion?

Desde la version 6.0 de Android, Google modifico la manera en la que un programador tiene que pedir permisos al usuario para realizar ciertas tareas o acceder a ciertas partes de Android como puede ser la conexion WiFi o la conexion Bluetooth.
Una de esas modificaciones que hizo Google fue que si un programador queria acceder a la conexion WiFi para poder escanear redes, tenia que pedir al usuario permisos de ubicacion, esto se hizo asi, segun Google, para que el usuario fuera totalmente consciente de que la aplicacion requeria un permiso "delicado" o de riesgo ya que una aplicacion normal no necesitaria nunca escanear las redes WiFi al alcance.

No obstante, WPS Wifi Checker Pro solamente necesita el permiso de ubicacion para poder acceder a las funciones WiFi que se encargan unica y exclusivamente de escanear redes y no usa, recaba informacion, ni hace absolutamente nada con la ubicacion del usuario ni el GPS ni nada parecido.

¬øCOMO FUNCIONA?

WPS Wifi Checker Pro tiene una amplia base de datos interna con los pins WPS por defecto de una gran cantidad de routers y los algoritmos generadores de dichos pins de muchos otros. Estos pins son conocidos y si el router tiene la configuracion de fabrica y no se ha configurado correctamente para desactivar la conexion por WPS o se ha cambiado el pin por defecto, es muy probable que dicho router sea vulnerable y que WPS Wifi Checker Pro pueda conectarse a dicha red usando esa vulnerabilidad.
